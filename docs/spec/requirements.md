# 要件

## 機能要件

### F1. サービス選択 / ログイン
- F1.1 サイドメニューに対応サービスのアイコン一覧を表示する。
- F1.2 サービス選択で、当該サービスのトップ/ログインページを内蔵 WebView に表示する。
- F1.3 Cookie・セッションはサービスごとに隔離し永続化する（`persist:<serviceId>`）。
- F1.4 ログイン状態を確認できる（`checkAuth`）。
- F1.5 セッション破棄（ログアウト相当）ができる。

### F2. ダウンロード設定
- F2.1 対象クリエイターを選択（未指定なら支援中すべて）。
- F2.2 ファイル種別フィルタ（image / video / audio / file）。
- F2.3 同時ダウンロード数の指定。
- F2.4 重複スキップの ON/OFF。

### F3. 一斉ダウンロード
- F3.1 支援中コンテンツを列挙し、まとめてダウンロードする。
- F3.2 進捗（合計/完了/スキップ/失敗）をリアルタイム表示する。
- F3.3 実行中にキャンセルできる。
- F3.4 失敗したファイルは記録し、再実行で再取得できる。

### F4. 重複検知
- F4.1 `サービス/ユーザー/年/月/投稿ID/ファイル` をキーに、取得済みを判定する。
- F4.2 メタデータ DB と実ファイルの両方を確認する。

### F5. ファイルビューワー
- F5.1 `サービス → ユーザー → 年 → 月 → 投稿ID` のツリーで一覧する。
- F5.2 投稿フォルダを OS のファイラで開ける。
- F5.3 （将来）画像/動画のインラインプレビュー。

### F6. 設定
- F6.1 ダウンロード保存先ルートの変更。
- F6.2 既定の同時ダウンロード数。

## 非機能要件

- N1. **ローカル完結**: 対象サイト以外への通信を行わない。
- N2. **セキュリティ**: `contextIsolation: true` / `nodeIntegration: false`。
  renderer は `window.api`（typed IPC）経由でのみ特権操作を行う。
- N3. **クロスプラットフォーム**: Windows を主対象。macOS/Linux も Electron で対応可能に保つ。
- N4. **耐障害性**: ダウンロード中断・失敗時もメタデータ整合を保ち、再開できる。
- N5. **拡張性**: 新規サイト追加は `Service` 実装 + 登録のみで完結する。
- N6. **パフォーマンス**: 大量投稿でもメモリを圧迫しない（将来はストリーミング DL、
  [roadmap](../roadmap.md) 参照）。
